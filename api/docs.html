Inventory

Up 
get /add_ingredient_qty.php
adds a quantity of ingredient to the user's inventory (addIngredientQuantity)
Adds the given quantity of ingredient to the user's inventory. It can be added to either the fridge or shopping list. You must specify either an id or ingredient name.

Query parameters

type (required)
Query Parameter — which inventory to add the quantity to
id (optional)
Query Parameter — ingredient id
name (optional)
Query Parameter — ingredient name
qty (required)
Query Parameter — ingredient quantity
qty-type (required)
Query Parameter — quantity unit of measure

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

ok

400

invalid parameters given

403

invalid authorization

get /missing_ingredients.php
returns the missing ingredients needed for upcoming recipes (getMissingIngredients)

post /create_recipe.php
creates a new recipe (createRecipe)
Creates a new recipe. You must specify a thumbnail as well as other required fields using a multipart form.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • multipart/form-data

Form parameters

thumbnail (optional)
Form Parameter — Image thumbnail
nombre (optional)
Form Parameter — Name of the recipe
dif (optional)
Form Parameter — Difficulty
visibility (optional)
Form Parameter — Visibility of the recipe. 0 = Private; 1 = Public
ingredients (optional)
Form Parameter — json representation of a list of ingredients (see Ingredient entity)
steps (optional)
Form Parameter — json representation of an array of steps (each step is a string)
tiempo (optional)
Form Parameter — Preparation time in minutes

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

ok

400

invalid input

403

invalid authorization

get /searchrecipe.php
searchs for recipes (seachRecipes)
Searchs for recipes given the name, ingredients and difficulty. If all fields are left empty, all recipes are returned.

Query parameters

nombre (optional)
Query Parameter — containing name
ingrediente (optional)
Query Parameter — containing ingredient
dificultad (optional)
Query Parameter — of difficulty

Return type

Recipe

Example data

Content-Type: application/json
{
  "tiempoelaboracion" : 60.0,
  "popularidad" : 250.0,
  "id_receta" : 1.0,
  "nombre" : "Macarrones con Tomate",
  "dificultad" : "Sencillo"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

returns recipes that match the criteria Recipe

403

get /schedule.php
returns scheduled recipes from the given date range (getSchedule)
Returns the recipes that were scheduled by the user for a date within the range given.

Query parameters

from (required)
Query Parameter — start of the date range
to (required)
Query Parameter — end of the date range

Return type

array[Schedule]

Example data

Content-Type: application/json
[ {
  "fecha" : "21-06-2020",
  "comida" : "Almuerzo",
  "id_receta" : 1.0,
  "nombre" : "Macarrones con Tomate"
}, {
  "fecha" : "21-06-2020",
  "comida" : "Almuerzo",
  "id_receta" : 1.0,
  "nombre" : "Macarrones con Tomate"
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json
